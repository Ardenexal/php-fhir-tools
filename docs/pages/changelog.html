Changelog

All notable changes to PHP FHIRTools are documented here.

The format is based on Keep a Changelog, and this project adheres to Semantic Versioning.

Version Format: MAJOR.MINOR.PATCH

  • MAJOR - Breaking changes
  • MINOR - New features (backwards compatible)
  • PATCH - Bug fixes (backwards compatible)

Version 0.5.0

Released: 2024-12-15

Added

  • FHIRPath evaluator with support for all standard functions
  • FHIRPath operators: comparison, boolean, math, collection
  • OperationOutcome error handling with FHIRPath expressions
  • Terminology service client for tx.fhir.org integration
  • Web Components for interactive demos
  • Service Worker for offline support
  • Support for FHIR R5 resources
  • php-wasm integration for browser-based PHP execution
  • Jekyll documentation site with search

Changed

  • Improved serialization performance by 30%
  • Updated to FHIR R4B specification
  • Enhanced validation error messages with FHIRPath locations
  • Refactored model generation for better code quality

Fixed

  • Fixed date/time parsing in FHIRPath evaluator
  • Corrected Bundle entry processing order
  • Fixed memory leak in large resource serialization
  • Resolved namespace conflicts in generated models

Security

  • Added input sanitization for FHIRPath expressions
  • Implemented rate limiting for terminology service calls

Version 0.4.0

Released: 2024-09-20

Added

  • FHIR Bundle support with transaction processing
  • XML serialization and deserialization
  • Validation modes: strict, lenient, none
  • Model generation command for CLI
  • Support for custom extensions
  • Contained resources support
  • Reference resolution helpers

Changed

  • Improved error messages with detailed context
  • Optimized memory usage for large bundles
  • Updated dependencies to latest versions

Fixed

  • Fixed choice type serialization (e.g., value[x])
  • Corrected cardinality validation
  • Fixed timezone handling in dateTime fields

    • Deprecated

    • setValidationMode() method (use constructor parameter instead)

    Fixed

  • Corrected Resource.meta serialization
